Описание реализованных функций

    GET /slots – возвращает список заездов. Поддерживает опциональные параметры from и to (даты в формате ISO). Если не указаны, по умолчанию берутся слоты на ближайшие 7 дней. Данные берутся из внутреннего хранилища (имитация бэкенда).

    GET /slots/{slot_id} – возвращает полную информацию о конкретном заезде.

    POST /bookings – создаёт бронирование.

        Проверяет, что слот существует и доступен.

        Атомарно уменьшает количество свободных картов и прокатной экипировки.

        При недостатке мест возвращает HTTP 409 (Conflict) с пояснением.

        В случае успеха возвращает объект бронирования.

    DELETE /bookings/{booking_id} – отменяет бронирование.

        Проверяет, что бронь принадлежит текущему пользователю.

        Запрещает отмену, если до старта осталось менее 10 минут (возвращает 400).

        Возвращает освободившиеся места обратно в слот.

    POST /ratings – позволяет клиенту оценить маршала после заезда.

        Проверяет, что заезд уже прошёл (по времени старта).

        Сохраняет оценку и комментарий (в реальности – в БД).

Дополнительно реализован эндпоинт GET /profile для получения профиля пользователя (с отметкой «постоянный клиент»).
Аутентификация и авторизация

Для простоты используется заглушка: клиент передаёт заголовок Authorization: Bearer user1 (где user1 – идентификатор существующего пользователя). В реальном проекте здесь была бы полноценная JWT-валидация.
Запуск сервера
bash

python main.py

Сервер запустится на http://localhost:8000. Документация Swagger доступна по адресу /docs.
Примеры запросов (через curl)

Получение списка слотов:
bash

curl -X GET "http://localhost:8000/slots"

Бронирование:
bash

curl -X POST "http://localhost:8000/bookings" \
  -H "Authorization: Bearer user1" \
  -H "Content-Type: application/json" \
  -d '{"slotId":"<slot_id>","participantsCount":2,"needRentalEquipment":true}'

Отмена:
bash

curl -X DELETE "http://localhost:8000/bookings/<booking_id>" \
  -H "Authorization: Bearer user1"

Оценка:
bash

curl -X POST "http://localhost:8000/ratings" \
  -H "Authorization: Bearer user1" \
  -H "Content-Type: application/json" \
  -d '{"bookingId":"<booking_id>","rating":5,"comment":"Отлично!"}'
